## Contributing

To contribute to a new release:
1. Ensure all changes are tested and approved.
2. Create a new release branch from `main`.
3. Tag the release with semantic versioning (e.g., `v1.0.0`).
4. Push the release artifacts to this repository.
5. Update the changelog and release notes.
